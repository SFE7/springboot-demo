│  ÉTAPE 3: CONFIGURER RENDER (2 minutes)                         │
└─────────────────────────────────────────────────────────────────┘

1. Allez sur https://render.com
2. Cliquez "Sign Up" ou "Log In"
3. Connectez votre compte GitHub
4. Cliquez "New +" (en haut à droite)
5. Sélectionnez "Web Service"
6. Trouvez votre dépôt et cliquez "Connect"

7. Configuration:
   ┌──────────────────────────────────────────┐
   │ Name:         spring-boot-demo           │
   │ Region:       Frankfurt (EU Central)     │
   │ Branch:       main                       │
   │ Environment:  Docker ⚠️ IMPORTANT        │
   │ Plan:         Free                       │
   └──────────────────────────────────────────┘

8. Cliquez "Create Web Service"


┌─────────────────────────────────────────────────────────────────┐
│  ÉTAPE 4: ATTENDRE & TESTER (8-10 minutes)                      │
└─────────────────────────────────────────────────────────────────┘

⏳ Render va automatiquement:
   1. Cloner votre dépôt GitHub
   2. Détecter le Dockerfile
   3. Télécharger Maven + Java 21
   4. Télécharger les dépendances Spring Boot
   5. Compiler votre projet
   6. Créer l'image Docker
   7. Démarrer l'application
   8. Vous donner une URL publique

✅ Une fois "Live", vous recevrez une URL comme:
   https://spring-boot-demo-XXXX.onrender.com

🧪 Testez ces URLs:
   /                    → Page d'accueil
   /person/list         → Liste des personnes
   /person/add          → Ajouter une personne
   /person/edit/0       → Modifier une personne
   /person/delete/0     → Supprimer une personne

Problème: Build failed
  → Allez dans "Logs" sur Render
  → Cherchez l'erreur en rouge
  → Corrigez et git push à nouveau

Problème: "Environment must be Docker"
  → Settings → Environment → Docker
  → Save Changes → Manual Deploy

Problème: App lente au démarrage
  → Normal! Mise en veille après 15 min
  → 30-60s pour réveiller l'app

Problème: 404 Not Found
  → Vérifiez que l'app démarre (logs)
  → Testez d'abord "/" puis "/person/list"

Problème: 502 Bad Gateway
  → L'app a crashé
  → Vérifiez les logs Spring Boot
  → Cherchez les exceptions Java
